# Spec: packages/ui

## Purpose
Next.js web UI. Two roles: observe the system (jobs, logs) and drive it (submit feedback, register MCPs). The UI itself is subject to the Ouroboros loop — user feedback can change the UI.

## Stack
- Next.js 14 (app router)
- Tailwind CSS
- No heavy component libraries — keep it lean so the meta-agent can modify it easily
- Server-sent events for live log/job streaming
- Default port: 7702

## Pages

### `/` — Dashboard
- Active jobs count, recent logs (last 20), system health (Redis connected, meta-agent lock held)
- Quick task submission input
- Live updates via SSE or 5s polling

### `/jobs` — Job List
- Full job table: id (truncated), backend, target, status, created, duration
- Click job → expand to show last 50 output lines
- Status badges: pending (grey), running (yellow), completed (green), failed (red)

### `/feedback` — Feedback / Self-Evolution
- Textarea + submit button
- Submits to `POST /api/feedback` → pushes to `ouro:feedback` queue
- Shows last 10 submitted feedbacks with timestamps
- Warning banner: "Feedback triggers code changes to Ouroboros itself"

### `/mcp` — MCP Registry
- List of registered MCPs from `ouro:mcp:registry`
- Form: name + connection string → `POST /api/mcp/register`
- Connection string examples shown (redis://, pg://, file://, github://)
- Each MCP shows: name, scheme, registered timestamp, status

### `/logs` — Live Logs
- SSE stream from `ouro:logs` Redis list
- Auto-scroll to bottom
- Filter by source (meta-agent, worker, gateway, mcp-factory)

## API Routes

```
POST /api/feedback     body: { text: string }   → push to ouro:feedback
POST /api/mcp/register body: { name, connectionString } → push to ouro:mcp:registry
GET  /api/jobs         → list jobs from ouro:jobs hash
GET  /api/logs         → SSE stream of ouro:logs
GET  /api/health       → { redis: bool, metaAgentLock: string|null }
```
