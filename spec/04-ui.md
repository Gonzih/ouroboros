# Spec: packages/ui

## Purpose
Observe the system (jobs, logs) and drive it (submit feedback, register MCPs). Evolves via the Ouroboros loop — user feedback directly modifies this file. Must stay simple enough for the meta-agent to modify.

## Stack

Same as cc-agent-ui — no framework, no build step:
- **Single `index.html`** — all HTML + CSS + JS in one file
- **Vanilla JS** — no React, no Vue, no Next.js. DOM manipulation directly.
- **Node.js HTTP server** (`server.js`) — serves static files + handles API routes
- **WebSocket** (ws library) — for live log streaming and job updates
- **Redis** (ioredis/redis) — server reads directly, pushes updates via WebSocket
- **JetBrains Mono** font, dark terminal aesthetic matching cc-agent-ui
- Default port: **7702**

Rationale: a single HTML file is trivially modifiable by the meta-agent. No transpilation, no bundler, no dependency on a JS framework that may change APIs. The Ouroboros loop needs to be able to edit the UI with a text editor and have it work immediately.

## Pages / Views (single-page, tab-based)

### Dashboard tab
- Active jobs count, last 5 log lines, meta-agent lock status (PID or "not running")
- Quick task input: text field + "Run task" button + backend selector (git/local)
- Quick feedback input: text field + "Send feedback" button (triggers Ouroboros loop)

### Jobs tab
- Table: job ID (8 chars), backend, target (truncated), status badge, created, elapsed
- Click row → expand inline to show last 100 output lines (WebSocket live if running)
- Status badges colored: pending=grey, running=amber, completed=green, failed=red, cancelled=dim

### Logs tab
- Live scrolling log from `ouro:logs` via WebSocket
- Filter chips by source: all | meta-agent | worker | gateway | mcp-factory
- Auto-scroll toggle (defaults on)

### Feedback tab
- Large textarea + "Submit" button
- Warning: "This triggers code changes to Ouroboros itself. You will be notified to verify."
- Recent feedback history (last 10 entries with timestamps and status: pending/applied/rejected)

### MCP tab
- List of registered MCPs: name, scheme, registered timestamp, operational status (green/red)
- "Register new" form: name field + connection string field + "Register" button
- Connection string hint examples shown inline
- Each MCP has a "Test" button → triggers validation via mcp-factory

## WebSocket Protocol

Server pushes JSON messages:
```typescript
type WsMessage =
  | { type: 'log'; entry: LogEntry }
  | { type: 'job_update'; job: Partial<Job> }
  | { type: 'job_output'; jobId: string; line: string }
  | { type: 'notify'; text: string }  // Ouroboros loop notifications
```

Client sends:
```typescript
type WsCommand =
  | { type: 'subscribe_job'; jobId: string }
  | { type: 'unsubscribe_job'; jobId: string }
```

## HTTP API (server.js routes)

```
POST /api/feedback    body: { text }              → push FeedbackEvent to ouro:feedback
POST /api/task        body: { instructions, backend, target } → push to ouro:tasks
POST /api/mcp/register body: { name, connectionString }      → forward to mcp-factory
GET  /api/jobs        → array of Job objects from ouro:jobs:*
GET  /api/health      → { redis: bool, metaAgentPid: string|null, jobCount: number }
```

## Authentication

None for v1. Future: corporate SSO via OpenID Connect / OAuth2 — single env var `OURO_OIDC_ISSUER` switches it on. Not in v1 scope.
